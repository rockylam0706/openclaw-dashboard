# OpenClaw Dashboard v1.0 - 产品需求文档 🦞

**版本：** v1.0（开发版）  
**创建日期：** 2026-03-06  
**状态：** 🟢 开发中  
**技术栈：** Node.js + React + Vite + Tailwind CSS

---

## 📋 需求背景

**痛点：**
- 查看 OpenClaw 状态需要终端输入 `openclaw-cn status`，麻烦
- 无法判断 AI 状态：工作中/卡死/待命
- 询问状态可能打断 AI 正在进行的工作

**目标：** 实时监控状态的可视化窗口，随时查看，无需打断

---

## 🎯 核心需求（P0）

| ID | 需求 | 描述 | 验收标准 |
|----|------|------|----------|
| **R1** | **网关状态监控** | 实时显示网关运行状态 | 状态准确，实时更新 <1s |
| **R2** | **任务状态监控** | 显示当前任务（做什么/待命） | 任务描述清晰，状态准确 |
| **R3** | **任务计时器** | 显示任务运行时长，判断卡死 | >5 分钟警告，>10 分钟告警 |
| **R4** | **TODO 列表** | 显示待完成任务规划 | 与 AI 记忆同步 |
| **R5** | **MD 文档编辑** | 在线编辑 SOUL.md、USER.md、MEMORY.md 等 | 查看 + 修改，保存成功 |
| **R6** | **Cron 管理** | 可视化添加/编辑 Cron 任务 | CRUD 操作完整 |
| **R7** | **快捷命令** | 一键执行 openclaw-cn 命令 | 带说明 + 二次确认 |
| **R8** | **响应式设计** | PC+ 移动端适配，支持公网 | 主流设备测试通过 |
| **R9** | **UI 设计** | 视觉效果优秀，易交互 | 美观现代，符合参考风格 |
| **R10** | **性能优化** | 长时间运行内存占用低 | <100MB，连续运行 24h |
