## 🐛 问题 1：版本号不更新

### 症状

用户报告："顶部和底部的版本号还是 2.0.4，没有实时跟随版本显示真实的版本号"

**位置**：
- Header 组件（页面顶部）
- App 组件 Footer（页面底部）

### 根因

版本号**硬编码**在 JSX 中：

```jsx
// Header.jsx - 第 12 行
<span>v2.0.4</span>

// App.jsx - 第 242 行
<p>🦞 OpenClaw Dashboard v2.0.4 | Made with ❤️</p>
```

每次发布新版本都需要手动修改代码，容易忘记更新。

### 修复方案

**创建动态版本号组件**：

1. **新增文件**：`src/components/Version.jsx`
   ```jsx
   import { useEffect, useState } from 'react';
   
   function Version() {
     const [version, setVersion] = useState('v2.0.7');
     
     useEffect(() => {
       // 从 package.json 读取真实版本
       fetch('/package.json')
         .then(res => res.json())
         .then(data => {
           if (data.version) {
             setVersion(`v${data.version}`);
           }
         })
         .catch(err => {
           console.warn('读取 package.json 失败，使用默认版本:', err);
         });
     }, []);
     
     return <span>{version}</span>;
   }
   
   export default Version;
   ```

2. **修改 Header.jsx**：
   ```jsx
   import Version from './Version';
   
   // 使用动态组件
   <span><Version /></span>
   ```

3. **修改 App.jsx**：
   ```jsx
   import Version from './Version';
   
   // Footer 中使用
   <p>🦞 OpenClaw Dashboard <Version /> | Made with ❤️</p>
   ```

### 验证

- ✅ Header 显示动态版本号
- ✅ Footer 显示动态版本号
- ✅ 修改 `package.json` 的 `version` 字段后自动更新
- ✅ 网络请求失败时使用默认版本（降级方案）

## 🐛 问题 2：消息排序错误

### 症状

用户报告："dashboard 的聊天窗口底部怎么会有几条 read\keen-har\cool-she 等消息，按时间戳他们不应该在这个位置才对"

**表现**：
- 旧的工具消息（如 `read`、`keen-har`）显示在最新消息位置
- 消息时间线混乱，不符合 chronological 顺序

### 根因

**时间戳格式不一致**导致排序失败：

```javascript
// 普通消息：数字时间戳（毫秒）
{
  "timestamp": 1772862898906  // ✅ 数字
}

// 工具消息：ISO 字符串
{
  "timestamp": "2026-03-07T05:52:34.582Z"  // ❌ 字符串
}
```

**排序逻辑**：
```javascript
// ❌ 错误：直接比较数字和字符串
const allMessages = [...messages, ...toolMessages]
  .sort((a, b) => a.timestamp - b.timestamp);
// 结果：字符串被转换为 NaN，排序失败
```

**数据验证**：
```bash
curl http://localhost:18790/api/chat/webui-messages | jq '.toolMessages[-1]'
# 输出：
{
  "timestamp": "2026-03-07T05:52:34.582Z",  // 字符串
  "toolName": "tender-h"
}
```

### 修复方案

#### 方案 1：前端统一转换（快速修复）

**修改 ChatWindow.jsx**：

```javascript
const renderAllMessages = () => {
  const allMessages = [...messages, ...toolMessages].sort((a, b) => {
    // ✅ 修复时间戳格式不一致问题
    const timeA = typeof a.timestamp === 'string' 
      ? new Date(a.timestamp).getTime() 
      : a.timestamp;
    const timeB = typeof b.timestamp === 'string' 
      ? new Date(b.timestamp).getTime() 
      : b.timestamp;
    return timeA - timeB;
  });
  
  return allMessages.map((msg, index) => (...));
};
```

#### 方案 2：后端统一格式（彻底修复）

**修改 server/routes/chat.js**（3 个 API）：

```javascript
// ✅ 统一时间戳格式为数字（毫秒）
timestamp: entry.timestamp 
  ? (typeof entry.timestamp === 'string' 
      ? new Date(entry.timestamp).getTime() 
      : entry.timestamp) 
  : (msg.timestamp 
      ? (typeof msg.timestamp === 'string' 
          ? new Date(msg.timestamp).getTime() 
          : msg.timestamp) 
      : Date.now()),
```

**修改位置**：
1. `GET /api/chat/sessions/:sessionKey/history` - 第 196 行、208 行
2. `GET /api/chat/feishu-messages` - 第 275 行、287 行
3. `GET /api/chat/webui-messages` - 第 362 行、374 行

### 最终方案

**同时实施方案 1 + 方案 2**：

- ✅ **后端**：统一输出数字时间戳（源头修复）
- ✅ **前端**：兼容处理字符串时间戳（防御性编程）

### 验证

**修复前**：
```bash
curl -s http://localhost:18790/api/chat/webui-messages | jq '{

# 输出：

  "msg_type": "number",

}
```

**修复后**：
```bash

```

**消息排序验证**：
- ✅ 最新消息显示在底部

## 📝 经验教训

### 1. 避免硬编码

**错误做法**：
```jsx
<span>v2.0.4</span>  // ❌ 硬编码
```

**正确做法**：
```jsx
<Version />  // ✅ 动态读取
```

### 2. 统一数据格式

**问题**：
- 不同模块使用不同时间戳格式（数字 vs ISO 字符串）
- 合并排序时未考虑格式转换

**解决**：
- 后端统一输出格式（数字时间戳）
- 前端做防御性转换（兼容旧数据）

### 3. 自动化版本管理

**建议**：
- CI/CD 流程中自动更新版本号
- 使用 Git tag 自动推导版本号
- 避免手动修改代码中的版本号
